## 1. なぜ複数の言語を学ぶのか

プログラミング言語は、仕事の内容や目的によって向き不向きがあります。

- 速く実行したい
- 文章として読みやすくしたい
- 既存のライブラリを使いたい
- 実験データを処理したい

このような要求が異なるため、1つの言語だけで全てを完璧に扱うのは難しいことがあります。

---

## 2. プログラムはどのように動くのか

ソースコードは、最終的にコンピュータが理解できる形式に変換されて実行されます。

### 代表的な実行方式

- コンパイラ方式
  - 事前に機械語へ変換する
  - 実行速度が高い傾向にある
  - 例: Fortran, C, C++
- インタプリタ方式
  - 実行時に1行ずつ解釈する
  - 直感的に書きやすい
  - 例: Python
- JIT方式
  - 実行時に必要に応じて最適化する
  - 高い生産性と速度の両立を目指す
  - 例: Julia

---

## 3. 代表的な言語の特徴

### Fortran

- 数値計算の歴史が長い
- 配列演算に強い
- 大規模な数値計算コードと相性が良い
- 科学技術計算で長く使われてきた

### Python

- 読みやすく、学習しやすい
- 科学計算・データ解析・機械学習向けのライブラリが豊富
- 試作や自動化に向いている
- ただし、単純なループ処理では性能が落ちやすい

### Julia

- Pythonに近い記述性と、Fortran/Cに近い実行性能を目指す
- 数値計算と科学技術計算を同じ言語で扱いやすい
- 新しいアルゴリズムの試作と本番計算の両方に向く
- 研究開発の現場で注目されている

---

## 4. どの言語を選ぶべきか

選択の基準は、主に次の3つです。

1. 何を実現したいか
2. どれだけ速く動かす必要があるか
3. 誰とコードを共有するか

### 使い分けの目安

- 既存の数値計算コードを扱う: Fortran
- データ解析・可視化・自動化: Python
- 新しいモデルやアルゴリズムを試しながら高速化したい: Julia

---

## 5. まとめ

プログラミング言語は、単なる「書き方」の違いではなく、実行方式・開発体験・性能の違いを含んだ道具です。

- 目的に応じて言語を選ぶ
- 実行方式の違いを理解する
- 研究・開発・教育の場面で、適切な言語を使い分ける

学習の第一歩は、1つの言語を深く理解したうえで、他の言語との違いを比較することです。
